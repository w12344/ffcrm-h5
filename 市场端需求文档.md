## 🎯 需求概述

### 1.1 背景与问题

**业务背景**
- 市场部的核心职责是获取高质量线索并高效下发给销售，但以往市场与销售数据割裂，市场只管“量”（留资数），销售只看“单”（成交额）。
- 市场负责人需要对 **CPA（平均获客成本）** 和 **CPS（单客成交成本）** 负责，必须追踪线索的全生命周期（五池四率）。
- 现有的线索录入、分配与追踪过程零散，缺少一个能统揽“大盘数据-渠道下钻-销售接单效能-单点线索分析”的一体化工作台。

**用户痛点**
- **痛点1：转化断层**：市场只知道给了多少线索，却不知道各渠道的真实成交率，无法优化投放策略。
- **痛点2：执行黑盒**：线索下发后，销售是否及时建联？激活率多高？市场无法实时监控，导致高价线索浪费。
- **痛点3：录入效率低**：各类渠道（小红书、抖音等）表格字段不一，手动导入查重难、效率低。

**机会点**
- 打通“看板-流转-管理”链路，将数据可视化（KPI、漏斗、渠道分析表格）与业务动作（批量导入、智能分配、跟进监控）高度集成。

### 1.2 需求范围

**本期包含**
- ✅ 市场核心 KPI 概览（线索数量、质量与流转、成交与转化）
- ✅ 核心市场指标分析大盘（五池四率漏斗、四率监控预警、获客成本 CPA/CPS）
- ✅ 渠道转化分析弹窗（支持从渠道下钻到对应销售人员的转化效能）
- ✅ 线索可视化气泡池（区分纯线索、客户、成交与入学池）
- ✅ 线索全维度跟进分析浮层（展示线索流转时间轴、通话/微信记录）
- ✅ 统一线索录入引擎（手动单条录入 + Excel/CSV 批量查重导入）
- ✅ 市场动作指令下发（线索管理跳转、智能分配、转交顾问）

## 🔄 业务数据闭环链路

市场端的业务逻辑形成了一个完美的**数据流转闭环**：
1. **获取（入水）**：通过 `LeadEntryModal` 录入/导入线索（小红书、抖音等） -> 记录成本与渠道。
2. **分配（分流）**：通过 `MarketActionToolbar` 批量/智能分配给顾问 -> 线索进入 `PoolCarousel`（线索池）。
3. **跟进（转化）**：顾问接单并沟通 -> 状态同步回市场端（建联、激活、成交、入学）。
4. **监控（仪表盘）**：市场通过 `MarketCoreMetricsModal` 监控五池四率，发现“建联率”或“激活率”过低触发预警。
5. **归因（下钻）**：点击 KPI 或大盘进入 `MarketChannelAnalysisModal`，下钻发现某渠道（或某销售）效能极低。
6. **优化（闭环）**：点击气泡池查看具体 `CustomerDetailModal` 录音/记录，定位问题后调整分配策略或叫停投放。

## 🎨 核心页面与功能设计

### 4.1 核心页面说明

**页面1：市场端首页 (`index.tsx`)**
- **核心元素**：顶部动态 KPI 卡片、中央业务水池（气泡流转）、右侧 AI 助手侧边栏。
- **交互逻辑**：
  - 点击 KPI 卡片 -> 打开对应指标的【渠道转化分析详表】。
  - 点击顶部 DashboardHeader -> 打开【核心市场指标分析】。
  - 点击右上角“线索管理” -> 跳转至完整的线索管理列表。
  - Hover/点击气泡 -> 打开【客户全维度跟进分析】。

**页面2：核心市场指标分析弹窗 (`MarketCoreMetricsModal`)**
- **维度一：线索转化维度**：
  - **五池四率漏斗图**：线索池 -> 客户池 -> 升级池 -> 成交池 -> 入学池。
  - **核心四率预警卡片**：建联率（<60%预警）、激活率（<20%预警）、成交率（<15%预警）、入学率（<95%预警）。带原因分析（如：执行力不足、交付风险高）。
  - **成本看板**：总曝光量、获客总成本、平均获客成本 (CPA)、成单单客成本 (CPS)。
- **维度二：渠道效能维度**：
  - **线索质量趋势**：近7天线索有效率折线图。
  - **渠道成交率排行**：各渠道成交转化率条形图。
  - **获客渠道占比**：来源占比饼图。

**页面3：渠道转化分析详表 (`MarketChannelAnalysisModal`)**
- **核心元素**：树状层级数据表格（Tree Table）。
- **交互逻辑**：
  - 父级展示**渠道**（如：微信公众号、抖音、转介绍），子级展示该渠道下**分配给的具体销售**（如：销售-张伟、销售-李娜）。
  - **核心字段**：新增线索数、已建联数、建联率、高意向数、高意向占比、成交数、成交率、带来营收。
  - **闭环意义**：解决“渠道背锅”还是“销售不行”的归因问题（例如抖音线索整体差，但张伟建联率77%，王强只有70%）。

**页面4：线索/客户详情分析浮层 (`CustomerDetailModal`)**
- **差异化展示模式**：
  - **纯线索模式 (Clue Mode)**：强调“高意向/重点跟进”标签、线索停留时长（小时）、录入时间。沟通记录展示为**电话沟通拨打记录**（时长/未接通）。时间轴仅显示“线索录入 -> 待分配”。
  - **客户/成交模式 (Customer/Enrolled Mode)**：强调“交付满意度/质量分”、“成交转化周期/意向度”。沟通记录展示为**深度微信沟通对话**。时间轴显示全链路流转状态。

**页面5：统一线索录入引擎 (`LeadEntryModal`)**
- **双轨录入机制**：
  - **手动单条录入**：支持详细的线索字段收集。具备严格的表单校验（如姓名、电话、线索来源为必填项）。包含渠道来源的一级枚举控制、跟进状态与接通状态的标准化标签，以及输入框字数限制，确保入库数据格式高度统一。
  - **批量文件导入**：支持 `.xlsx`, `.xls`, `.csv` 格式文件的拖拽或点击上传，并提供标准导入模板下载，规范外部导入数据的表头。
- **防重与导入历史看板**：
  - **防重逻辑**：后端及上传解析时，基于“电话”或“微信号”进行全局查重，重复数据将自动跳过或提示覆盖。
  - **历史导入记录清单**：展示导入时间、文件名称、总条数、成功导入数（绿色高亮）、重复跳过数（黄色警告）、操作人及状态，确保所有的批量操作皆可追溯、防漏防错。

## 🔧 详细设计与数据字段字典

为了确保前端展示的数据与后端系统、业务动作完全对齐，以下详细说明本工作台中所有核心指标、业务池及字段的来源、计算规则及更新机制。

### 7.1 核心 KPI 动态卡片处理逻辑

**卡片 1：市场获客规模（看线索总盘）**
- **功能名称**：线索数量卡
- **功能描述**
  - 用于展示市场端最核心的线索入库底座，回答“当前周期内一共拿到了多少潜在生源”。
- **处理逻辑**
  1. 主指标展示：**线索数量**（即总分配线索数）。
  2. 统计维度按当前选定的时间周期统计（默认为本月）。
  3. 若目标存在，则展示目标完成率（即：线索完成率 = 实际线索数量 / 线索目标数量 * 100%）。没设置目标则目标展示0。
  4. 若同比存在，则展示同比数值增长或者下降值，增长数值前面显示+，下降数值前面显示-。
  5. 右侧图表区展示主目标的环形进度条。
  6. 子指标固定展示两组：
     - 待处理线索（分配后未建联）及变动趋势。
     - 今日新增线索（T+0 实时新增）及变动趋势。
  7. 交互：点击卡片支持下钻至对应的渠道转化分析弹窗。

**卡片 2：线索有效性与流转（看激活与下发质量）**
- **功能名称**：线索质量与流转卡
- **功能描述**
  - 用于展示市场给到销售后的真实承接情况，不允许仅看分配数，回答“下发的线索里到底有多少是真实有效的”。
- **处理逻辑**
  1. 主指标展示：**线索建联率**（百分比形式）。
  2. 公式口径：线索建联率 = 客户池总数 / 线索池总数 * 100%。
  3. 若同比存在，则展示同比数值增长或者下降值，增长数值前面显示+，下降数值前面显示-。
  4. 右侧图表区展示线索建联率的环形进度条。
  5. 子指标固定展示两组：
     - 已建联线索（绝对值人数）及变动趋势。
     - 高意向占比（升级池总数 / 客户池总数 * 100%）及变动趋势。

**卡片 3：最终产出与 ROI（看转化结果）**
- **功能名称**：线索成交与转化卡
- **功能描述**
  - 用于展示市场获取的线索最终的变现能力，回答“这些流量最终为公司带来了多少单和多少钱”。
- **处理逻辑**
  1. 主指标展示：**成交转化率**（百分比形式）。
  2. 公式口径：成交转化率 = 成交池总数 / 升级池总数 * 100%。
  3. 若同比存在，则展示同比数值增长或者下降值，增长数值前面显示+，下降数值前面显示-。
  4. 右侧图表区展示成交转化率的环形进度条。
  5. 子指标固定展示两组：
     - 已成交线索数（绝对值单数）及变动趋势。
     - 已成交线索销售总额（该批成交客户绑定的实收金额之和）及变动趋势。

### 7.2 五池四率核心指标定义与数据来源
市场端最核心的数据仪表盘（Dashboard），其数据来源需跨越市场、销售、财务与教务系统。

| 指标/业务池 | 业务定义 | 数据来源与触发节点 | 计算公式/取值逻辑 |
| --- | --- | --- | --- |
| **线索池 (Leads)** | 市场部获取到的所有潜在客户名单 | 市场专员通过 `LeadEntryModal` 手动录入/Excel导入，或通过第三方开放平台API自动抓取（如抖音/小红书表单留资）。 | `COUNT(录入系统且未作废的线索)` |
| **客户池 (Customers)** | 销售接单并成功完成首次有效建联的线索 | 销售在工作台认领/被分配后，产生第一条**有效沟通记录**（如电话接通时长>10秒或微信双向回复）。 | `COUNT(具备有效沟通记录的客户)` |
| **升级池 (Upgrades)** | 被判定为高意向、有望近期成交的潜客 | 销售手动更改客户意向评级（如设为A级/极高意向），或系统AI助手基于语义和跟进频次自动判定。 | `COUNT(意向评级≥A 或 星标≥4星的客户)` |
| **成交池 (Deals)** | 成功支付定金或全款签约的客户 | 财务系统/订单系统确认首笔收款，或合同审核通过。 | `COUNT(关联了已支付订单的客户)` |
| **入学池 (Enrollments)**| 实际完成报到并开始上课的客户 | 教务系统排课状态变更为“已排课/已报到”。 | `COUNT(教务状态=报到的客户)` |
| **建联率** | 衡量线索真实有效性与销售跟进速度 | 实时计算。 | `客户池数量 / 线索池数量 * 100%` |
| **激活率** | 衡量线索质量与销售培育跟进能力 | 实时计算。 | `升级池数量 / 客户池数量 * 100%` |
| **成交率** | 衡量销售临门一脚转化能力 | 实时计算。 | `成交池数量 / 升级池数量 * 100%` |
| **入学率** | 衡量交付质量，预判退费风险 | 实时计算。 | `入学池数量 / 成交池数量 * 100%` |

### 7.2 成本与ROI指标数据映射
此类数据主要用于市场管理层评估投放渠道 ROI。

| 指标名称 | 业务定义 | 数据来源 | 更新频率 |
| --- | --- | --- | --- |
| **总曝光量** | 广告在各渠道的展示次数 | 对接巨量引擎/磁力引擎/腾讯广告等第三方开放API拉取，或由市场部每日手工填报。 | T+1 每日更新 |
| **获客总成本** | 市场投放总消耗金额 | 广告账户余额消耗数据接口同步，结合线下活动报销审批系统数据。 | T+1 每日更新 |
| **CPA(平均获客成本)** | 单个线索的平均获取价格 | 系统实时聚合计算：`获客总成本 / 线索池总数量`。 | 随总成本和线索数变动 |
| **CPS(成单单客成本)** | 单个成交客户的营销成本 | 系统实时聚合计算：`获客总成本 / 成交池总数量`。 | 随总成本和成交数变动 |

### 7.3 渠道分析详表字段映射 (`MarketChannelAnalysisModal`)
用于追溯具体流量阵地和对应承接销售的效能。

| 字段名称 | 界面层级 | 字段来源与判定规则 | 业务意义 |
| --- | --- | --- | --- |
| **渠道名称** | 父级行 | 线索录入时选择的 `source` 字段（如小红书、抖音、转介绍等）。 | 用于横向对比不同流量阵地的质量与转化。 |
| **销售姓名** | 子级行 | 线索分配表 `allocation_records` 中的 `advisor_name`。 | 用于明确同一渠道下，不同销售的承接与逼单能力差异。 |
| **新增线索数** | 基础数据 | 筛选对应时间段内，`source=当前渠道` 且 `advisor_id=当前销售` 的线索总数。 | 衡量流量分配是否均匀。 |
| **带来营收** | 转化结果 | 筛选上述线索中，已流转至“成交池”客户所绑定的**实收订单金额总和**。 | 衡量该渠道的直接财务回报。 |

| 字段名 (Key) | 界面名称 | 数据类型 | 必填 | 校验规则/长度限制 | 默认值/可选项 (枚举值) | 业务流转与说明 |
| --- | --- | --- | --- | --- | --- | --- |
| `name` | 姓名 | String | **是** | 最大长度 50 字符 | - | 市场手动录入或表单授权抓取。客户的基础标识。 |

| `wechat` | 微信号orQQ | String | 否 | 最大长度 50 字符 | - | 备用建联方式，部分微信域线索可能仅有此项。 |

| `familyPhone2` | 家里人手机2 | String | 否 | 11位数字正则校验 | - | 备用联系方式2。 |

### 7.5 关键交互控制逻辑

**线索分配与转交逻辑 (`AssignAdvisorModal`)**
- 支持针对单个或批量线索执行【智能分配/指定转交】。
- **派单算法（智能分配）**：系统根据当前在线顾问的工作负载（手中待处理线索数）、转化历史得分（该渠道转化率），加权后分配给最优销售。
- **回调机制**：分配成功后，需即时回调更新水池状态（移除未分配线索，或变更气泡颜色），并触发前端气泡动画重绘。

**UI 动态伸缩逻辑 (`ChatAssistant` 侧边栏)**
- 右侧 AI 助手支持无极拖拽改变宽度（`20% - 60%`）。
- 宽度变化时，左侧气泡水池和图表区自动使用 `flex: 1` 填补剩余空间，通过 ResizeObserver 确保图表自适应重绘，不破坏视觉布局。
